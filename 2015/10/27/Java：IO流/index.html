Java的IO

发表于 2015-10-27   |   分类于 Java   |  

有一个blog总结的很好，说的是IO流可以按照三种方式来区分：

  1. 按照功能分：输入流与输出流
  2. 按照数据传输单位分类：字节流与字符流
  3. 按照功能分类：节点流与过滤流
  • 流：在程序中所有的数据都是以流的方式进行传输或保存的，程序需要数据的时候要使用输入流读取数据，而当程序需要将一些数据保存起来的时候，就要使用输出流完成。

字节流与字符流

从根本来看，Java的IO分为了字节流与字符流，下面是一张经典的图阐述了Java中各种IO类的关系：

简单来说，字符流是以char[]为单位进行读写；而字节流是以byte[]进行读写；字符流与字节流之间的桥梁是InputStreamReader与OutputStreamWriter

因为字节流与字符流方法类似（只是读取与写入的数据一个是byte，另一个是char），这里选取字节流做详细的分析

过滤流

过滤流不能直接跟文件打交道，只能通过节点流进行相关的操作。可以从其构造方法中看出：

常用的过滤流有缓冲输出流BufferedOutputStream

该类实现缓冲的输出流。通过设置这种输出流，应用程序就可以将各个字节写入底层输出流中，而不必针对每次字节写入调用底层系统。具体内容可参见：磁盘的IO读取方式

当缓冲区写满或者关闭输出流时，一次性输出到流，或者调用flush()方法主动将缓冲区输出到流。

Scanner读取数据

Scanner 是 java.util 包里的。它一个可以使用正则表达式来分析基本类型和字符串的简单文本扫描器。

Scanner 使用分隔符模式将其输入分解为标记，默认情况下该分隔符模式与空白匹配。然后可以使用不同的 next 方法将得到的标记转换为不同类型的值。
