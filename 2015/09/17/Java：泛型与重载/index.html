Java重载

说道java的方法重载，不得不提方法签名（这是用来区分不同方法的标识符）。由于重载方法之间的方法名是相同的，那么我们必须找到唯一标示方法的签名——jvm认为只需要形参列表和返回值就可以唯一确定一个方法，也就是说，方法签名=形参列表+方法名。所以正确的重载应该只有以下两个条件：

  1. 方法名相同
  2. 参数列表不同，即参数类型、个数、顺序至少有一项不相同

而返回值相同or不同，方法访问限定符相同or不同并不会对重载构成什么影响。

泛型

再来看泛型，Java的泛型是在编译阶段确定的。而在编译时，编译器会对泛型进行一个操作：就是将泛型的类型参数擦除（也就是类型擦除）。什么意思呢？对于编译器而言，List<Object>与List<String>在编译后都会变成List，其他的泛型附加信息对于JVM都是不可见的。

依据上言，对于JVM而言，

  1. public List<String> convert(List<Intger> list)====>public List convert(List list)
  2. public List<Integer> convert(List<Intger> list)====>public List convert(List list)

这两个方法的名字、参数列表一致，所以编译器不认为这是重载，报错。

类型擦除带来的特性

  1. 泛型类没有自己独有的Class类对象。意味着，并不存在List<String>.class或者List<Integer>.class，只有List.class存在
  2. 静态变量被泛型类的所有实例共享。意味着，MyClass<T>中的静态变量staticVar无论在MyClass<Integer>或者MyClass<String>都通过MyClass.staticVar访问
  3. 泛型的类型参数不能用在Java异常处理的catch语句中。因为异常处理是由JVM在运行进行的。由于类型信息被擦除，JVM是无法区分两个异常类型MyException<String>和MyException<Integer>的。对于JVM来说，它们都是 MyException类型的。也就无法执行与异常对应的catch语句。
